Step 1 — Add a Redirect URI to your Entra App Registration

The MSAL pop-up requires a redirect URI pointing back to your app. You can use the same app registration as the Worker (SecureAuth-EmailChecker).

  • Go to portal.azure.com → Entra ID → App registrations → SecureAuth-EmailChecker
  • Left sidebar → Authentication → + Add a platform → Single-page application
  • Redirect URI: https://jerrya-byte.github.io/secure-auth-spa/
  • Tick Access tokens and ID tokens under Implicit grant
  • Click Configure → Save
⚠ The redirect URI must exactly match your GitHub Pages URL including the trailing slash.

Step 2 — Paste credentials into the MSAL config panel below

You need the same Tenant ID and Client ID from your app registration. Paste them into the MSAL SSO Configuration panel below.

Step 3 — Existing User username matching

After SSO the app extracts the username from the Entra UPN — the part before the @ symbol. This must match a username in your user_credentials table.

ℹ Example: SSO returns UPN [email protected] → username extracted as john.doe → looked up in user_credentials table.
